What is CANopen Safety?

CANopen Safety is defined in CiA 304 and provides a black channel safety layer on top of standard CANopen networks. It enables SIL3/PLe safety communication using standard CAN hardware, making it cost-effective for safety-critical applications in industrial automation, mobile machinery, and transportation.

AZIT CANopen Safety Esclave Stack

Our CANopen Safety Esclave implementation provides a complete, certified solution for integrating safety functions into your CANopen devices. Built on 30+ years of embedded systems expertise, it features a modular architecture that works with any standard CANopen slave stack.

Black Channel Architecture

The black channel architecture ensures safety communication even over potentially unsafe transmission media. This provides:

  • Independent safety layer over standard CANopen
  • Works with any CAN controller hardware
  • Simplified certification process
  • Cost-effective safety solution

Initialization Example

C 
/* CANopen Safety Esclave initialization */
#include "canopen_safety_slave.h"

COSS_CONFIG config = {
    .node_id = 0x10,
    .vendor_id = 0x12345678,
    .product_code = 0x1000,
    .safe_inputs = 8,      /* bytes */
    .safe_outputs = 8      /* bytes */
};

/* Initialize CANopen Safety slave */
if (coss_init(&config) == COSS_OK) {
    /* Start safety communication */
    coss_start();
}

Cyclic Processing

C 
/* Call in your cyclic task */
void safety_task(void) {
    uint8_t safe_inputs[8];
    uint8_t safe_outputs[8];

    /* Read safe inputs from your application */
    read_safe_inputs(safe_inputs);

    /* Process CANopen Safety communication */
    coss_process(safe_inputs, safe_outputs);

    /* Write safe outputs to your application */
    write_safe_outputs(safe_outputs);
}
