Available Training Programs

CANopen Training

Duration: 2-3 days

Comprehensive CANopen training covering protocol fundamentals, device profiles, configuration, and practical implementation.

  • CAN and CANopen fundamentals
  • Object dictionary and communication
  • Network management (NMT, Heartbeat)
  • SDO, PDO, Emergency, Sync
  • Device profiles (CiA 301, 401, 402)
  • Configuration and commissioning
  • Hands-on exercises with real devices

EtherCAT & FSoE Training

Duration: 2-3 days

In-depth training on EtherCAT real-time Ethernet and FSoE safety protocol for industrial automation applications.

  • EtherCAT fundamentals and architecture
  • Telegram processing and timing
  • CoE (CANopen over EtherCAT)
  • Distributed clocks synchronization
  • FSoE safety protocol
  • Network configuration and tuning
  • Practical lab sessions

J1939 Training

Duration: 2 days

Focused training on SAE J1939 protocol for heavy-duty vehicle and off-highway equipment applications.

  • J1939 protocol overview
  • Parameter groups (PGNs) and SPNs
  • Transport protocol (BAM, CMDT)
  • Address claiming
  • Diagnostics (DM messages)
  • J1939-71 vehicle application layer
  • Practical exercises and tools

Protocol Gateway Development

Why Protocol Gateways

Modern industrial systems often need to integrate devices using different protocols. Protocol gateways enable seamless communication between incompatible networks, allowing you to integrate legacy equipment with modern systems or connect different automation islands.

Common Gateway Scenarios

  • CANopen to EtherCAT gateway
  • J1939 to CANopen gateway
  • Proprietary to standard protocol gateway
  • Modbus to CANopen gateway
  • Multi-protocol gateway (3+ protocols)
  • Safety to standard network gateway

Gateway Features

  • Bidirectional data transfer
  • Configurable data mapping
  • Real-time performance
  • Web-based configuration
  • Diagnostic capabilities
  • Status monitoring
  • Firmware updates

Gateway Development Process

  1. Requirements Analysis
    Define protocols, data mappings, performance requirements, and features
  2. Architecture Design
    Design gateway architecture, hardware platform selection, software architecture
  3. Implementation
    Develop gateway firmware, configuration tools, and user interfaces
  4. Testing & Validation
    Comprehensive testing with real devices and networks
  5. Documentation & Delivery
    Complete documentation, user manuals, and commissioning support
