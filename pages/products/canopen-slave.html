AZIT CANopen Slave Stack

Our CANopen Slave stack provides a complete, production-ready implementation of the CiA 301 standard for industrial automation devices. Built with 30+ years of embedded expertise, it offers robust communication in CANopen networks.

Key Applications

  • Industrial sensors and actuators
  • Motor drives and motion controllers
  • I/O modules
  • Encoders and measurement devices
  • HMI and display units

Object Dictionary

Includes a flexible Object Dictionary (OD) implementation with support for manufacturer-specific objects, configuration parameters, and process data mapping. OD generation tools provided.

Key Benefits

  • CiA 301 v4.2 compliant
  • No third-party dependencies
  • Low hardware dependency
  • Low OS dependency
  • Minimal footprint
  • OD configuration tools

Quick Start Integration

Initialization Example

C 
/* CANopen Slave initialization */
#include "canopen_slave.h"

CO_CONFIG config = {
    .node_id = 10,
    .bitrate = CO_BITRATE_250K,
    .heartbeat_time = 1000  /* ms */
};

/* Initialize CANopen stack */
if (co_slave_init(&config) == CO_OK) {
    /* Start communication */
    co_slave_start();
}

Cyclic Processing

C 
/* Call in your main loop */
void main_task(void) {
    while(1) {
        /* Process CANopen communication */
        co_slave_process();

        /* Application code */
        update_application_data();

        delay_ms(1);
    }
}
