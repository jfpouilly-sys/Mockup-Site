AZIT FSoE Master Stack

Our FSoE Master implementation provides a complete, certified solution for safety controllers managing FSoE slave devices in EtherCAT networks. Designed for industrial safety PLCs, motion controllers, and safety logic solvers requiring SIL3/PLe certification.

Key Applications

  • Safety PLCs and controllers
  • Motion control systems with safety functions
  • Industrial robots with safety zones
  • Machine safety control systems
  • Emergency stop controllers

White Channel Architecture

Like our FSoE Slave stack, the master uses white channel architecture, allowing independence from the underlying EtherCAT master stack. This provides flexibility in choosing your EtherCAT master implementation while maintaining full safety certification.

Quick Start Integration

The AZIT FSoE Master stack is designed for straightforward integration into safety controllers.

Initialization Example

C 
/* FSoE Master initialization */
#include "fsoe_master.h"

/* Configure master and connections */
FSOE_MASTER_CONFIG master_config = {
    .max_connections = 4,
    .cycle_time = 10  /* ms */
};

/* Initialize FSoE master */
if (fsoe_master_init(&master_config) == FSOE_OK) {
    /* Add slave connections */
    FSOE_CONNECTION_CONFIG conn = {
        .connection_id = 0x0001,
        .watchdog_time = 100,
        .safe_inputs = 8,
        .safe_outputs = 8
    };
    fsoe_master_add_connection(&conn);
}

Cyclic Processing

C 
/* Call in your cyclic safety task */
void safety_controller_task(void) {
    uint8_t safe_outputs[8];
    uint8_t safe_inputs[8];

    /* Read local safety inputs */
    read_local_safety_inputs(safe_outputs);

    /* Process all FSoE connections */
    fsoe_master_process();

    /* Get data from specific slave */
    fsoe_master_get_slave_data(0x0001, safe_inputs);

    /* Process safety logic */
    process_safety_logic(safe_inputs, safe_outputs);
}
