CANopen Safety Master Solution

The AZIT CANopen Safety Master provides complete control and monitoring of CANopen Safety networks according to CiA 304. Designed for safety controllers and PLCs, it manages multiple safety slaves and provides comprehensive diagnostic and configuration capabilities.

Key Capabilities

Our master implementation handles all aspects of safety network management including SSDO communication for configuration, SRDO for cyclic safety data exchange, and comprehensive error handling and diagnostics.

Master Features

  • Manage up to 127 safety slaves
  • Parallel safety connections
  • SSDO configuration services
  • SRDO cyclic data exchange
  • Network diagnostics and monitoring
  • Safety state management

Quick Start Integration

The AZIT CANopen Safety Master stack provides comprehensive API for managing safety networks.

Initialization Example

C 
/* CANopen Safety Master initialization */
#include "canopen_safety_master.h"

COSM_CONFIG config = {
    .node_id = 0x01,
    .max_slaves = 16,
    .vendor_id = 0x12345678,
    .product_code = 0x2000
};

/* Initialize CANopen Safety master */
if (cosm_init(&config) == COSM_OK) {
    /* Start safety master */
    cosm_start();
}

Adding Safety Slaves

C 
/* Add and configure a safety slave */
COSM_SLAVE_CONFIG slave_cfg = {
    .node_id = 0x10,
    .safe_inputs = 8,
    .safe_outputs = 8,
    .watchdog_time = 100  /* ms */
};

/* Add slave to network */
if (cosm_add_slave(&slave_cfg) == COSM_OK) {
    /* Configure and activate slave */
    cosm_configure_slave(0x10);
    cosm_activate_slave(0x10);
}
